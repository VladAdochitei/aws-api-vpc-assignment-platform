## Communication Style
- **Brevity:** Use brief, clear expressions. Prefer concise over comprehensive.
- **Directness:** Provide recommendations and decisions, not exhaustive options.

## Coding Philosophy
- **Simplicity over efficiency:** Value code that is easy to maintain and recall over clever optimizations or complex patterns. Simple code is preferred even if it requires slightly more lines.
- **Incremental development:** Guide the project evolution step-by-step. Do not speculate about future requirements or design ahead unless explicitly asked.
- **Python-first:** All backend systems should be built in Python.

## Working Dynamic
- User guides the evolution of the project one piece at a time
- Do not propose large architectural changes unprompted
- Focus on the immediate task at hand
